## 二、性能优化（P1 — 影响大规模生成）

### 2.1 邻居搜索 O(n²) → O(n log n)
**文件**：`generators/height_assign.py:69-79`

当前 `_find_neighbors()` 对每个地块线性扫描全部地块，500 地块 = 25 万次距离计算 × 3 轮平滑。

**修复方案**：
```python
def _build_grid(plots, cell_size=100.0):
    grid = {}
    for idx, plot in enumerate(plots):
        cx, cy = plot["center"]
        gx, gy = int(cx // cell_size), int(cy // cell_size)
        grid.setdefault((gx, gy), []).append(idx)
    return grid

def _find_neighbors_grid(idx, plots, grid, radius):
    cx, cy = plots[idx]["center"]
    cell_size = radius
    gx, gy = int(cx // cell_size), int(cy // cell_size)
    neighbors = []
    for dx in (-1, 0, 1):
        for dy in (-1, 0, 1):
            for j in grid.get((gx + dx, gy + dy), []):
                if j == idx:
                    continue
                ox, oy = plots[j]["center"]
                if ((cx - ox) ** 2 + (cy - oy) ** 2) ** 0.5 < radius:
                    neighbors.append(j)
    return neighbors
```

预期收益：500 地块从 250K → 约 5K 次计算（50 倍提升）。

### 2.2 街区提取循环去重 O(n²) → O(1)
**文件**：`generators/block_extract.py:68-69`

```python
# 当前：每次重建完整列表做线性查找
if cycle_key not in [tuple(sorted(c)) for c in cycles]:
```

**修复**：使用 `set` + 规范化表示
```python
seen_cycles = set()

# 规范化：从最小节点 ID 开始，选择字典序较小的方向
def canonical_cycle(path):
    min_idx = path.index(min(path))
    rotated = path[min_idx:] + path[:min_idx]
    reversed_rot = [rotated[0]] + rotated[1:][::-1]
    return tuple(rotated) if tuple(rotated) < tuple(reversed_rot) else tuple(reversed_rot)

cycle_key = canonical_cycle(path)
if cycle_key not in seen_cycles:
    seen_cycles.add(cycle_key)
    cycles.append(path)
```

### 2.3 建筑实例化（去重 Mesh）
**文件**：`generators/batch_buildings.py:100`

当前每栋建筑创建独立 Mesh，500 栋 = 500 个 Mesh 对象。

**修复方案**：按 typology + sections 分组，同组共享 Mesh 引用：
```python
mesh_cache = {}

for idx, spec in enumerate(building_specs):
    key = (typology, params["sections"])
    if key not in mesh_cache:
        mesh_cache[key] = _generate_mesh_only(params)  # 只生成 mesh 数据
    
    obj = bpy.data.objects.new(f"CityP_City_Building_{idx}", mesh_cache[key])
```

预期收益：500 栋建筑从 500 个 Mesh 降至约 10-20 个 Mesh（内存节省 95%+）。

### 2.4 LOD 简化：bmesh 替代 bpy.ops
**文件**：`generators/lod_system.py:65-71`

`bpy.ops.mesh.dissolve_limited` 在循环中调用最多 20 次，每次涉及模式切换、上下文检查、undo 推送。

**修复**：
```python
import bmesh

bm = bmesh.new()
bm.from_mesh(mesh)
bmesh.ops.dissolve_limit(
    bm,
    angle_limit=0.1,
    verts=bm.verts,
    edges=bm.edges,
)
bm.to_mesh(mesh)
bm.free()
```

预期收益：单次 LOD 操作从秒级降至毫秒级。

### 3.4 高度模型升级为双峰分布

当前 `height_assign.py` 使用单点线性衰减。设计文档要求"双峰分布（CBD + 副中心）"：

```python
def assign_heights_dual_peak(dist_ratio, cbd_weight=0.7, sub_center_dist=0.5):
    cbd_height = lerp(max_height, min_height, dist_ratio)
    sub_height = lerp(max_height * 0.6, min_height, abs(dist_ratio - sub_center_dist) * 2)
    return cbd_height * cbd_weight + sub_height * (1 - cbd_weight)
```
