> 版本：v1.0 · 截至 2026-04-04 · 覆盖全部 23 个源文件

## 一、算法管线总览

城市白膜的生成是一个**五层管线**，每层输出作为下一层输入：

```
[输入] → 路网生成 → 街区提取 → 地块分割 → 高度分配 → 建筑生成 → [输出]
         (L1)         (L2)        (L3)        (L4)        (L5)
```

两种输入模式共享 L2-L5 管线：
- **OSM 模式**：Overpass API → 解析 → 墨卡托投影 → 直接得到 Footprint → 跳 L1-L3
- **程序化模式**：参数 → L1 路网 → L2 街区 → L3 地块 → L4 高度 → L5 建筑

## 二、各层算法详解

### L1 · 路网生成

**文件**：`generators/city_layout.py`（89 行）

**核心数据结构**：`RoadGraph` — 邻接表表示的平面图
```
RoadGraph
├── nodes: [(x, y), ...]          # 节点坐标列表
├── edges: [(a, b), ...]          # 边（节点索引对）
└── adj:   {nid: {nid, ...}}      # 邻接表（set 去重）
```

**算法流程**：

```
1. 铺设正交网格
   范围: [-radius, radius]²
   间距: main_spacing (默认 300m)
   节点数: (2 * radius / spacing + 1)²

2. 节点扰动
   每节点: (x, y) += uniform(-δ, +δ)
   δ = main_spacing × perturbation_pct / 100
   默认 δ = 300 × 15% = 45m

3. 连接主干边
   每个节点连接右邻居和上邻居 → 保证网格连通性

4. 次级细分
   对每个网格单元：
   - 在单元中心附近添加子节点
   - 连接子节点到四个角
   → 将 300m 格细分为 4 个 ~150m 子格

5. 连通性清理
   移除度 < 2 的节点（悬挂节点）
   → 保证 DCEL 面提取不会失败
```

**复杂度**：O(n) — n 为网格节点数

**关键参数**：

| 参数 | 默认值 | 作用 |
|------|--------|------|
| main_spacing | 300m | 主干道间距 |
| sub_spacing | 90m | 次级路网间距（通过细分间接控制） |
| perturbation_pct | 15% | 有机扰动幅度 |

### L2 · 街区提取（DCEL 右转法）

**文件**：`generators/block_extract.py`（123 行）

**算法**：从路网图的边集提取有界面集（城市街区）

**核心思想**：从一条路段出发，每到路口都尽量**右转**，走回到起点为止，圈出来的就是一个城市街区。

**算法流程**：

```
for start_node in all_nodes:
    for start_neighbor in sorted(adj[start_node]):
        if edge(start_node, start_neighbor) already visited:
            continue

        path = [start_node, start_neighbor]
        current = start_neighbor
        prev = start_node

        while current != start_node:
            incoming_angle = atan2(current - prev)

            best_next = None
            best_right_turn = -π

            for nb in sorted(adj[current]):
                if nb == prev or edge(current, nb) used:
                    continue

                edge_angle = atan2(nb - current)
                turn = edge_angle - incoming_angle
                right_turn = -turn  # 正值 = 右转

                if right_turn > best_right_turn:
                    best_right_turn = right_turn
                    best_next = nb

            if best_next is None:
                break  # 死路，不是有效环路

            path.append(best_next)
            mark edge(current, best_next) as visited
            prev = current
            current = best_next

        if current == start_node and len(path) >= 3:
            # 找到有效环路 → 提取多边形
            polygon = [nodes[nid] for nid in path]
            if area(polygon) > 100m²:
                blocks.append(polygon)
```

**角度归一化**：
```python
while turn_angle > π:  turn_angle -= 2π
while turn_angle < -π: turn_angle += 2π
```

**已知问题**：
- 去重使用 `tuple(sorted(path))`，丢失了路径的邻接关系和方向信息
- 两个不同环路若经过相同节点集会被误判为重复
- 复杂度 O(n² × L)，L 为环路平均长度

**复杂度**：O(n² × L) — 最坏情况

### L3 · 地块分割（退线 + 递归轴向分割）

**文件**：`generators/plot_split.py`（118 行）

**算法流程**：

```
输入: 街区多边形 (block_polygon)
输出: 建筑地块列表 (plots)

1. 退线（Setback）
   计算多边形质心 (cx, cy)
   对每个顶点:
     dist = distance(vertex, centroid)
     if dist < setback:
       vertex → centroid  (收缩到质心)
     else:
       scale = (dist - setback) / dist
       vertex → centroid + (vertex - centroid) × scale

2. 递归轴向分割
   function split(polygon, depth):
     if depth > 10 or area(polygon) <= max_area × 1.3:
       add polygon to plots
       return

     axis = random.choice([X, Y])
     split_pos = min_val + (max_val - min_val) × uniform(0.3, 0.7)

     poly_a, poly_b = split_polygon_by_line(polygon, axis, split_pos)

     if both valid:
       split(poly_a, depth + 1)
       split(poly_b, depth + 1)
     else:
       add polygon to plots
```

**多边形分割算法**（Sutherland-Hodgman 变体）：

```
function split_polygon_by_line(polygon, axis, pos):
    poly_a = []  # axis 坐标 >= pos 的部分
    poly_b = []  # axis 坐标 < pos 的部分

    prev = polygon[-1]
    for curr in polygon:
        prev_side = sign(prev[axis] - pos)
        curr_side = sign(curr[axis] - pos)

        # 分类当前顶点
        if curr_side >= 0: poly_a.append(curr)
        else:              poly_b.append(curr)

        # 边跨越分割线 → 计算交点
        if curr_side != prev_side:
            t = (pos - prev[axis]) / (curr[axis] - prev[axis])
            intersection = prev + t × (curr - prev)
            poly_a.append(intersection)
            poly_b.append(intersection)

        prev = curr

    return (poly_a if len >= 3, poly_b if len >= 3)
```

**面积阈值**：
| 建筑类型 | 最小面积 | 最大面积 |
|---------|---------|---------|
| 塔楼地块 | 200m² | 1500m² |
| 综合体 | 3000m² | 8000m² |
| 老居民楼 | 200m² | 500m² |

**已知问题**：
- 退线使用径向收缩，对非凸多边形可能产生自交
- 分割线平行于边时产生退化顶点

### L4 · 高度分配（双变量模型）

**文件**：`generators/height_assign.py`（79 行）

**模型公式**：

```
height(i) = base_height(dist_ratio) + noise(i) × floor_variance × 3.0

base_height(t) = lerp(avg_floors × 3.0, 4.0, t)
               = avg_floors × 3.0 × (1 - t) + 4.0 × t

其中 t = min(distance_to_center / 800.0, 1.0)
```

**噪声场生成**（3 轮高斯平滑）：

```
1. 初始化: noise[i] ~ N(0, 1)  for each plot i

2. 平滑 (3 轮):
   for each plot i:
     neighbors = {j | distance(center_i, center_j) < 100m}
     if neighbors:
       noise'[i] = 0.5 × noise[i] + 0.5 × mean(noise[j] for j in neighbors)
     else:
       noise'[i] = noise[i]

3. 应用:
   height[i] = base_height(dist_ratio_i) + noise[i] × floor_variance × 3.0
   height[i] = max(height[i], 3.0)
```

**类型学分类**（委托给 `typology_classifier`）：

```
if OSM tags available:
  → 查 typologies.json 的 osm_type_mapping
  → 单候选直接返回，多候选随机选择
else:
  → 启发式规则:
    height > 60 且 area < 800   → stepped_tower
    height > 40 且 area > 1000  → podium_tower
    height > 15 且 400 < area < 2000 → slab 或 tapered
    height < 20 且 area < 300   → old_res
    area > 1500 且 height > 15  → complex
    height < 12 且 area > 500   → industrial
    其他 → 按 dist_ratio 随机分配
```

**性能瓶颈**：`_find_neighbors()` 为 O(n²)，500 地块 = 250K 次距离计算 × 3 轮

### L5 · 建筑生成（BMesh 拉伸框架）

**文件**：`generators/building_gen.py`（284 行）

**核心几何原语**（4 个函数）：

| 函数 | 操作 | 用途 |
|------|------|------|
| `make_rect_face` | 创建矩形面 | 建筑底面 |
| `extrude_face` | 沿 Z 轴拉伸面 | 生成楼层 |
| `scale_face_xy` | 以面中心为基准缩放 XY | 步退（setback） |
| `rotate_face_z` | 以面中心为基准绕 Z 旋转 | 扭转（twist） |

**7 种建筑形态生成器**：

#### 1. Stepped Tower（步退式高层）

```
输入: base_w, base_d, height, sections, setback_ratio, twist_deg

高度权重: [N, N-1, ..., 1]  →  底段最高，顶段最矮
总权重: N(N+1)/2
第 i 段高度: H × (N-i) / [N(N+1)/2]

for i in 0..N-1:
    extrude_face(section_height[i])
    if i < N-1:
        setback_x = setback_ratio + uniform(-variance, +variance)
        setback_y = setback_ratio + uniform(-variance, +variance)
        scale_face_xy(setback_x, setback_y)
        if twist != 0:
            rotate_face_z(twist × (-1)^i)  # 交替方向
        add_floor_loop(inset=0.3)
```

**设计意图**：递减权重避免"积木感"；XY 独立收缩打破对称；交替扭转避免"拧毛巾"效果。

#### 2. Tapered Tower（线性锥削塔）

```
输入: base_w, base_d, height, taper_ratio

steps = 12
per_step_ratio = taper_ratio^(1/steps)  # 等比数列

for _ in 0..steps-1:
    extrude_face(H/steps)
    scale_face_xy(per_step_ratio, per_step_ratio)
```

#### 3. Podium + Tower（裙房 + 塔楼）

```
输入: base_w, base_d, height, podium_height, tower_ratio

阶段 1: 裙房
    extrude_face(podium_height)
    add_floor_loop(inset=0.5)
    scale_face_xy(tower_ratio, tower_ratio)

阶段 2: 塔楼（3 段步退，同 Stepped Tower 子算法）
    tower_height = height - podium_height
    weights = [3, 2, 1]
    for i in 0..2:
        extrude_face(tower_section_height[i])
        if i < 2:
            setback + twist
```

#### 4. Slab Block（板楼）

```
输入: base_w, base_d, height

floor_height = 3.0
floors = height / 3.0

for i in 0..floors-1:
    extrude_face(3.0)
    if i % 2 == 0:
        add_floor_loop(inset=0.15)  # 每隔一层加腰线
```

#### 5. Old Residential（老式居民楼）

```
输入: base_w, base_d, height

w = base_w × uniform(0.7, 1.0)  # 随机缩小
d = base_d × uniform(0.7, 1.0)

for i in 0..floors-1:
    extrude_face(3.0)
    if i == last_floor:
        inset_individual(thickness=0.3, depth=-0.3)  # 屋顶内凹
```

#### 6. Mixed-Use Complex（综合体）

```
输入: base_w, base_d, height, complex_base_height

阶段 1: 大底盘
    extrude_face(base_height=18m)
    add_floor_loop(inset=0.5)

阶段 2: 多塔楼（2-4 座随机分布）
    for each tower:
        create_offset_rect(random_position, random_size, base_height)
        for j in 0..tower_floors-1:
            extrude_face(tower_step)
            if j < tower_floors-1:
                scale_face_xy(0.95, 0.95)  # 微锥削
```

#### 7. Industrial（工业厂房）

```
输入: base_w, base_d, height

阶段 1: 主体
    extrude_face(height × 0.7)

阶段 2: 锯齿屋顶
    n_teeth = base_w / 8.0
    tooth_width = base_w / n_teeth
    tooth_height = height × 0.3

    for i in 0..n_teeth-1:
        create_offset_rect(tooth_position, tooth_size, height × 0.7)
        extrude_face(tooth_height)
        # 单侧斜坡
        for v in top_verts:
            if v.x > 0:
                v.z += tooth_height × 0.5
```

## 三、辅助算法

### 3.1 墨卡托投影

**文件**：`utils/geo_projection.py`（31 行）

```
x = R × (lon - origin_lon) × cos(origin_lat)
y = R × (lat - origin_lat)

其中 R = 6371000m (地球半径)
角度单位: 弧度
```

批量版本预计算 `cos_lat`、`lon_rad`、`lat_rad` 避免重复计算。

### 3.2 OSM 解析

**文件**：`utils/osm_parser.py`（135 行）

- **JSON 解析**：遍历 `elements` 数组，分类为 node/way
- **XML 解析**：遍历 `<node>` 和 `<way>` 标签
- **建筑判别**：`building` tag 存在且值不为 `no/none/null`
- **道路判别**：`highway` tag 存在且排除 `footway/path/steps/pedestrian/cycleway` 等步行级道路
- **Footprint 提取**：按 way 的 node 引用列表查找坐标，自动闭合

### 3.3 类型学分类器

**文件**：`utils/typology_classifier.py`（138 行）

- **OSM 模式**：查 `typologies.json` 的 `osm_type_mapping` 表
- **启发式模式**：基于 height + area + dist_ratio 的规则树
- **参数生成**：从 `typologies.json` 读取各类型的参数范围，用 rng 采样

### 3.4 批量建筑放置

**文件**：`generators/batch_buildings.py`（170 行）

- 按 typology 查 `TYPOLOGY_CONFIG` 获取尺寸比例
- 从地块面积反推 base_w/base_d
- 调用 `building_gen.generate_building()` 逐栋生成
- 生成道路 Mesh（每段路 = 一个矩形面）
- 组织到 `CityP_Buildings` 和 `CityP_Roads` Collection

### 3.5 LOD 系统

**文件**：`generators/lod_system.py`

| LOD 级别 | 操作 |
|---------|------|
| 0 | 简化为包围盒（8 顶点 Box） |
| 1 | 面数降至 25%（`bpy.ops.dissolve_limited`） |
| 2 | 面数降至 50% |
| 3 | 不简化 |

### 3.6 屋顶细节

**文件**：`generators/detail_gen.py`

随机生成：
- **冷却塔**（60% 概率）：截顶锥体（8 顶点）
- **水箱**（50% 概率）：圆柱体（8 段）
- **电梯井**（40% 概率）：长方体

### 3.7 渲染管线

**文件**：`pipelines/` 下 4 个文件

| 管线 | 引擎 | 核心配置 |
|------|------|---------|
| Arch White | EEVEE | 白漫反射 + 柔光区域灯 + GTAO |
| Golden Hour | Cycles 256spp | 15° 低角太阳 + amber 色温 + Filmic |
| Neon Rain | EEVEE + Bloom | 自发光点光源 + 湿面低粗糙度 + 低曝光 |
| Iso Clear | EEVEE + Freestyle | 正交投影 35.26° + 描边 1.5px |

## 四、数据结构

### RoadGraph

```python
RoadGraph {
    nodes: List[Tuple[float, float]]   # 节点坐标 (x, y) 米
    edges: List[Tuple[int, int]]        # 边 (节点索引对)
    adj: Dict[int, Set[int]]            # 邻接表
}
```

### BuildingSpec

```python
BuildingSpec {
    "plot":       List[Tuple[float, float]],  # 地块多边形顶点
    "center":     Tuple[float, float],         # 地块中心
    "area":       float,                       # 地块面积 m²
    "height":     float,                       # 建筑高度 m
    "typology":   str,                         # 7 种类型之一
    "dist_ratio": float,                       # 距市中心距离比 [0, 1]
}
```

### OSMData

```python
OSMData {
    nodes:       Dict[int, {"lat", "lon", "tags"}]
    ways:        List[{"id", "nodes", "tags"}]
    buildings:   List[Way]                      # 过滤后的建筑 way
    highways:    List[Way]                      # 过滤后的道路 way
    origin_lat:  float                          # 投影原点纬度
    origin_lon:  float                          # 投影原点经度
}
```

## 五、算法复杂度汇总

| 层 | 算法 | 复杂度 | 瓶颈 |
|----|------|--------|------|
| L1 | 路网生成 | O(n) | 无 |
| L2 | 街区提取 | O(n² × L) | 循环去重 |
| L3 | 地块分割 | O(d × p) | d=递归深度, p=顶点数 |
| L4 | 高度分配 | O(n²) | 邻居搜索 |
| L5 | 建筑生成 | O(m × s) | m=建筑数, s=段数 |
| - | 坐标投影 | O(v) | v=顶点总数 |
| - | 批量放置 | O(m) | 逐栋生成（无实例化） |

**典型规模**：
- 程序化城市（半径 400m）：~50 节点 → ~20 街区 → ~100 地块 → ~100 建筑
- OSM 城市（半径 500m）：~500-2000 建筑 → 邻居搜索成瓶颈

## 六、算法设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 路网表示 | 邻接表 + 坐标列表 | 内存紧凑，遍历高效 |
| 街区提取 | 右转法（非 Shapely） | 零外部依赖 |
| 地块分割 | 递归轴向分割（非 Voronoi） | 可控性强，可预测 |
| 高度模型 | 线性衰减 + 高斯平滑 | 简单有效，3 轮平滑足够 |
| 建筑几何 | BMesh 拉伸（非 Geometry Nodes） | Python 可控，Blender 3.6+ 兼容 |
| 类型学 | JSON 配置 + 启发式规则 | OSM tags 不完整，规则兜底 |
| 渲染管线 | 程序化节点构建（非预设文件） | 零外部依赖，动态可调 |
