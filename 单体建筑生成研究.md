# 单体建筑生成研究：从建筑构成到参数化算法

> **目标**：研究建筑师设计单体建筑的思维过程，将后现代主义、构成主义、包豪斯、极简主义等风格的设计法则提炼为可编程的参数化规则，为 CityPhase 下一代单体建筑生成器提供理论基础和算法蓝图。

---

## 一、建筑师如何设计一个建筑

### 1.1 设计过程的五个层级

建筑师的思维过程通常遵循以下层级，从抽象到具体：

```
概念 (Concept) 
    ↓ 场地、功能、文化语境
体量 (Massing)
    ↓ 比例、方向性、虚实关系
构成 (Composition)
    ↓ 元素排列、韵律、对比
细部 (Detail)
    ↓ 材质、节点、光影
氛围 (Atmosphere)
    ↓ 尺度、情绪、叙事
```

当前 CityPhase v1.0.0 的生成器只做到了 **体量 (Massing)** 层级——即输入宽高、退台、扭转等参数，输出一个三维形体。但这只是建筑的"骨架"。真正让建筑具有"设计感"的是 **构成 (Composition)** 和 **细部 (Detail)** 层级的处理。

### 1.2 从"搭积木"到"写乐谱"

| 层级 | 当前做法 | 目标做法 |
|------|---------|---------|
| 体量 | 拉伸 + 缩放 + 布尔 | 保持 |
| 构成 | 无 | **核心增量：元素的组织法则** |
| 细部 | 简单的 inset/extrude | 风格化的细部语法 |
| 氛围 | 靠渲染管线 | 生成阶段预埋光影线索 |

**核心洞察**：建筑设计不是随机堆叠元素，而是遵循一套**构成语法 (Composition Grammar)**。就像音乐有和声规则，建筑也有其内部的"句法"。

## 二、四种建筑风格的设计法则拆解

### 2.1 包豪斯主义 (Bauhaus, 1919-1933)

**代表建筑师**：Walter Gropius, Mies van der Rohe, Marcel Breuer

#### 核心信条
- **形式追随功能 (Form follows function)**
- **少即是多 (Less is more)** — Mies
- 拒绝装饰，强调材料诚实表达
- 工业化生产与艺术的统一

#### 构成法则 → 算法映射

```
法则 1：直角网格系统
- 建筑由正交网格决定
- 开间 (bay) 尺寸固定：3.6m / 5.4m / 7.2m
- 生成算法：base_w = n × bay_width

法则 2：自由平面 (Free Plan)
- 结构骨架（柱网）与围护（墙体）分离
- 立面是"皮"，可以独立于结构自由布置
- 生成算法：先生成柱网框架，再套外皮

法则 3：流动空间 (Flowing Space)
- 水平方向的连续性 > 垂直分隔
- 空间之间用半高墙、平台、台阶过渡
- 生成算法：用阶梯式楼板替代平层

法则 4：材料诚实
- 钢结构 = 纤细的黑色线条
- 混凝土 = 粗野的灰色体量
- 玻璃 = 透明的虚空
- 生成算法：不同材质对应不同几何处理

法则 5：比例控制
- 黄金分割或√2 矩形系列
- 模数化设计
- 生成算法：所有尺寸为 base_unit 的整数倍
```

**算法蓝图：Bauhaus 生成器**

```python
def gen_bauhaus(params):
    bay = params["bay_width"]        # e.g. 5.4m
    nx = params["bays_x"]            # e.g. 4
    ny = params["bays_y"]            # e.g. 3
    floor_h = params["floor_height"] # e.g. 3.6m
    floors = params["floors"]
    
    # 1. 生成柱网骨架
    columns = generate_column_grid(nx, ny, bay, floor_h, floors)
    
    # 2. 自由平面：随机移除部分墙体创造流动空间
    walls = generate_walls_from_grid(columns)
    walls = random_remove(walls, ratio=params["openness"])
    
    # 3. 流动空间：阶梯式楼板
    if params["flowing_space"]:
        slabs = generate_stepped_slabs(columns)
    else:
        slabs = generate_flat_slabs(columns)
    
    # 4. 玻璃幕墙外皮
    skin = generate_curtain_wall(columns, transparency=params["glass_ratio"])
    
    # 5. 比例校验：所有尺寸为 bay 的整数倍
    assert all(dim % bay < 0.01 for dim in all_dimensions)
```

### 2.2 构成主义 (Constructivism, 1919-1930)

**代表建筑师**：El Lissitzky, Konstantin Melnikov, Vladimir Tatlin

#### 核心信条
- **建筑是空间中的构成 (Construction in space)**
- 动态对角线、悬挑、相交体块
- 建筑应像工程师的桥梁一样表达结构力量
- 拒绝古典对称，拥抱不对称张力

#### 构成法则 → 算法映射

```
法则 1：体块交叉与悬挑
- 两个或多个矩形体以非正交角度相交
- 一个体块从另个体块中"刺出"
- 生成算法：随机生成 2-4 个矩形体，
  赋予不同角度和偏移，执行布尔并集

法则 2：对角线动力
- 视觉重心沿对角线移动
- 楼梯、坡道、连廊成为构图主角
- 生成算法：在体块之间插入倾斜的
  连接元素（坡道、天桥）

法则 3：不对称平衡
- 左右不等重，但视觉达到动态平衡
- "重的元素靠近中心，轻的元素远离中心"
- 生成算法：用大体积低高度 vs 小体积高
  高度的组合创造力矩平衡

法则 4：工业材料表达
- 混凝土、钢、玻璃的原始质感
- 不加装饰，暴露结构节点
- 生成算法：粗糙倒角（chamfer 大半径）
  + 外露钢构细线

法则 5：色彩作为结构标记
- 红、黄、蓝三原色标记功能区域
- 色彩不是装饰，是"空间的语法标注"
- 生成算法：不同功能体块分配不同
  材质色（红=公共，蓝=私密，黄=交通）
```

**算法蓝图：Constructivist 生成器**

```python
def gen_constructivist(params):
    n_volumes = params["n_volumes"]  # 2-5
    volumes = []
    
    for i in range(n_volumes):
        vol = generate_random_box(
            w=rng(10, 30),
            d=rng(10, 30), 
            h=rng(15, 80)
        )
        # 法则 1：随机旋转，避免正交
        vol.rotate_z(rng(-30, 30))
        # 法则 3：不对称偏移
        vol.offset(x=rng(-20, 20), y=rng(-20, 20))
        volumes.append(vol)
    
    # 法则 1：布尔并集形成整体
    building = boolean_union(volumes)
    
    # 法则 2：插入对角线连接
    for i in range(n_volumes - 1):
        bridge = generate_diagonal_bridge(volumes[i], volumes[i+1])
        building = boolean_union(building, bridge)
    
    # 法则 5：色彩标记
    for i, vol in enumerate(volumes):
        color = ["red", "blue", "yellow"][i % 3]
        assign_material(vol, color)
```

### 2.3 极简主义 (Minimalism, 1960s-)

**代表建筑师**：Tadao Ando, John Pawson, Alberto Campo Baeza, Peter Zumthor

#### 核心信条
- **极致的减法**
- 光、影、材质、空间是唯一的"装饰"
- 精确到毫米的比例控制
- "空"比"满"更有力量

#### 构成法则 → 算法映射

```
法则 1：纯粹几何体
- 立方体、圆柱体、圆锥体等基本形
- 拒绝复杂曲线，只使用"柏拉图立体"
- 生成算法：建筑 = 1-3 个基本体的组合

法则 2：墙之诗 (Wall as protagonist)
- 墙不是边界，是雕塑
- 墙比建筑主体更厚、更高、更突出
- 生成算法：在主体外围生成 2-5 面
  独立的"片墙"，高度 > 主体高度

法则 3：光的切割
- 建筑是捕捉光的容器
- 窄缝窗、天窗、挑檐创造戏剧性光影
- 生成算法：在立面特定高度开水平窄缝
  在屋顶开天窗（inset + extrude 向下）

法则 4：材质的单一性
- 一个建筑只用 1-2 种材料
- 清水混凝土、白色灰泥、原木
- 生成算法：单一材质 + 微妙的粗糙度变化

法则 5：比例的极致精确
- 1:1, 1:2, 1:3, 2:3 的简单整数比
- 空间序列：大→小→大→小 的韵律
- 生成算法：所有尺寸基于一个基本模数
  的 1x, 2x, 3x, 1/2x, 1/3x
```

**算法蓝图：Minimalist 生成器**

```python
def gen_minimalist(params):
    module = params["module"]  # e.g. 3.6m
    
    # 法则 1：纯粹几何体
    main_box = generate_box(
        w=module * params["w_mod"],    # e.g. 3x = 10.8m
        d=module * params["d_mod"],    # e.g. 2x = 7.2m
        h=module * params["h_mod"]     # e.g. 4x = 14.4m
    )
    
    # 法则 2：片墙
    for _ in range(params["n_walls"]):
        wall = generate_wall(
            w=module * rng(2, 6),
            h=module * params["h_mod"] * rng(1.2, 1.8),
            t=0.3
        )
        wall.offset_from(main_box, distance=module * rng(1, 3))
        wall.rotate_y(rng(-5, 5))  # 微微倾斜
    
    # 法则 3：光的切割
    # 水平窄缝
    for level in params["slit_levels"]:
        slit = generate_horizontal_slit(
            building=main_box,
            z=module * level,
            height=0.15
        )
    
    # 天窗
    if params["skylight"]:
        skylight = generate_skylight(main_box, ratio=0.3)
```

### 2.4 后现代主义 (Postmodernism, 1960s-1990s)

**代表建筑师**：Robert Venturi, Michael Graves, Philip Johnson, James Stirling

#### 核心信条
- **少即是烦 (Less is a bore)** — Robert Venturi
- 建筑应该说话，应该隐喻，应该有趣
- 引用历史符号，但用现代手法重构
- 色彩、装饰、戏谑的回归

#### 构成法则 → 算法映射

```
法则 1：历史符号的抽象引用
- 拱券 → 抽象为圆角窗洞阵列
- 柱廊 → 抽象为垂直条纹立面
- 山花 → 抽象为顶部三角形装饰
- 生成算法：从历史元素库中随机选择
  1-3 个符号，以现代比例重新诠释

法则 2：矛盾与并置
- 古典元素 + 现代材料
- 严肃的比例 + 戏谑的色彩
- 生成算法：立面分成 2-3 个区域，
  每个区域使用完全不同的"语法"

法则 3：色彩作为建筑语言
- 不再是功能标记，是情感表达
- 粉红、薄荷绿、天蓝等高饱和度色
- 生成算法：立面分区 + 随机色彩分配

法则 4：装饰作为结构
- 装饰不是附加物，是结构的一部分
- 例如：柱子就是排水管，窗框就是通风口
- 生成算法：每个装饰元素必须同时
  具有功能意义（即使是假功能）

法则 5：规模的游戏
- 超大尺度的日常元素
- 例如：一个 10m 高的门洞
- 生成算法：将某个细部放大到
  建筑尺度（如巨型拱券贯穿全楼）
```

**算法蓝图：Postmodern 生成器**

```python
HISTORICAL_MOTIFS = {
    "arch":      gen_abstracted_arch,
    "column":    gen_abstracted_column_grid,
    "pediment":  gen_abstracted_pediment,
    "dome":      gen_abstracted_dome,
    " arcade":   gen_abstracted_arcade,
}

def gen_postmodern(params):
    base = generate_box(w=30, d=20, h=40)
    
    # 法则 1：引用历史符号
    motifs = random.sample(list(HISTORICAL_MOTIFS.keys()), k=params["n_motifs"])
    for motif in motifs:
        element = HISTORICAL_MOTIFS[motif](base, exaggeration=params["exaggeration"])
        base = boolean_union(base, element)
    
    # 法则 2：立面分区矛盾并置
    facade = split_facade_into_zones(base, n_zones=params["n_zones"])
    for zone in facade.zones:
        zone.style = random.choice(["grid", "strip", "arch", "plain"])
        zone.material = random.choice(PASTEL_PALETTE)
    
    # 法则 5：规模游戏
    if params["gigantic_element"]:
        element = random.choice(motifs)
        scale_factor = rng(3, 8)
        gigantify(element, scale_factor)
```

### 2.5 粗野主义 (Brutalism, 1950s-1970s)

**代表建筑师**：Le Corbusier (晚期), Alison & Peter Smithson, Paul Rudolph, Louis Kahn

#### 核心信条
- **混凝土是建筑的唯一语言**
- 纪念性尺度，压倒人性的体量
- 形式追随材料：模板的木纹、螺栓孔都保留
- 几何的纯粹性与力量的表达

#### 构成法则 → 算法映射

```
法则 1：巨型几何体
- 单一巨大形体，拒绝细碎
- 建筑像从地里长出来的巨石
- 生成算法：1-2 个超大体量，
  尺度是普通建筑的 2-5 倍

法则 2：雕塑式立面
- 立面不是平面，是三维雕塑
- 深凹的阳台、突出的楼梯间、倾斜的墙面
- 生成算法：在立面上进行"雕刻"操作
  （inset 深度大、extrude 幅度大）

法则 3：材料的粗粝表达
- 清水混凝土的模板痕迹
- 保留施工痕迹作为"诚实的装饰"
- 生成算法：立面添加 noise displacement
  模拟模板纹理

法则 4：光线的戏剧性
- 窄而深的开口，内部极暗
- 光线像刀一样切割空间
- 生成算法：小窗洞 + 深凹的窗框
  （inset 深度 = 1-2m）

法则 5：不对称的纪念性
- 非对称构图，但有强烈的轴向感
- 一个巨型元素主导，其他辅助
- 生成算法：确定一个"主角体块"（最大/最高），
  其他体块围绕它组织
```

## 三、构成语法的通用框架

### 3.1 建筑构成的"句法元素"

无论哪种风格，单体建筑的构成都可分解为以下原子元素：

```
元素层级：

0. 场地点 (Site Point)
   - 建筑的锚点

1. 体量 (Volume)
   - 盒体、圆柱、棱锥等基本形
   - 属性：w, d, h, offset(x,y,z), rotation(x,y,z)

2. 面操作 (Surface Operation)
   - inset：内凹（窗洞、阳台）
   - extrude：外凸（飘窗、雨棚、装饰）
   - boolean：切割、合并、差集
   - subdivide：细分面（格栅、幕墙分格）

3. 连接 (Connection)
   - 桥接：两个体块之间的通道
   - 过渡：阶梯、坡道、平台
   - 咬合：体块的穿插关系

4. 细部 (Detail)
   - 线脚 (molding)：顶部收边、底部勒脚
   - 柱子 (column)：独立支撑元素
   - 栏杆 (railing)：边界标记
   - 屋顶 (roof)：已有 6 种类型

5. 材质 (Material)
   - 色彩、粗糙度、透明度
   - 纹理（模板痕、砖缝、金属板缝）
```

### 3.2 构成法则的"谓词语法"

建筑设计可以表达为一组谓词操作序列：

```
PLACE(volume, position, rotation)
EXTRUDE(face, distance, taper=0)
INSET(face, thickness, depth)
BOOLEAN(op, target, tool)   # op ∈ {union, difference, intersect}
SUBDIVIDE(face, u, v)
CONNECT(vol_a, vol_b, type)  # type ∈ {bridge, ramp, stair}
MATERIAL(surface, mat_id)
ALIGN(ref_a, ref_b, axis)    # axis ∈ {x, y, z}
OFFSET(element, direction, distance)
```

**示例**：一个包豪斯式悬挑体块

```
PLACE(box_a, (0,0,0), 0)
  → w=20, d=15, h=12

PLACE(box_b, (10,0,8), 0)  
  → w=15, d=15, h=10
  
BOOLEAN(union, box_a, box_b)

# 底层架空
INSET(bottom_face_of_box_a, 2.0, 4.0)

# 玻璃幕墙外皮
EXTRUDE(facade_faces, 0.2, taper=0)
MATERIAL(facade_faces, "curtain_wall")
```

## 四、从研究到实现的路线图

### Phase 1: 构成语法引擎 (Composition Engine)

实现 `composition.py`，提供谓词语法的 Python API：

```python
class CompositionBuilder:
    def place(self, shape, pos, rot):
        """放置基本体"""
    
    def extrude(self, face_selector, distance, taper=0):
        """面外凸"""
    
    def inset(self, face_selector, thickness, depth):
        """面内凹"""
    
    def boolean(self, op, target, tool):
        """布尔运算"""
    
    def subdivide(self, face_selector, u, v):
        """面细分"""
    
    def connect(self, vol_a, vol_b, connection_type):
        """体块连接"""
    
    def apply_style(self, prs):
        """应用风格规则集"""
```

```
        minimalist_gen.py         # 极简主义生成器
        postmodern_gen.py         # 后现代生成器
        brutalist_gen.py          # 粗野主义生成器
        style_prs.py              # 风格参数集定义
```

### Phase 3: 混合风格与演变

允许风格之间的**插值 (interpolation)**：

```python
def gen_mixed_style(style_a, style_b, blend_ratio=0.5):
    prs_a = load_prs(style_a)
    prs_b = load_prs(style_b)
    mixed_prs = interpolate_prs(prs_a, prs_b, blend_ratio)
    return generate_from_prs(mixed_prs)
```

例如：`bauhaus × brutalist` = 包豪斯的比例控制 + 粗野主义的雕塑感

## 五、关键算法问题与解决方案

### 问题 1：体块布尔运算的稳定性
- **挑战**：Blender 的布尔运算在复杂相交时容易失败
- **方案**：
  1. 使用 `bmesh` 手动实现凸多面体的布尔并集（因为建筑体块通常是凸的或有限凸分解）
  2. 失败时退化为 "bounding box merge"（将相交体块合并为包围盒）

### 问题 2：风格一致性与随机性的平衡
- **挑战**：完全随机失去风格感，完全规则失去变化
- **方案**：
  1. 定义**风格不变量 (Style Invariants)**：无论什么随机种子都必须遵守的规则
  2. 定义**风格变量 (Style Variables)**：允许随机波动的参数范围
  3. 例：包豪斯的"正交网格"是不变量，"开间数量"是变量

### 问题 3：从 PRS 到 BMesh 的高效映射
- **挑战**：频繁的 `bmesh.ops` 调用性能开销大
- **方案**：
  1. 延迟求值：所有操作先记录为指令列表，最后一次性执行
  2. 几何缓存：相同参数的"面操作"复用缓存网格
  3. LOD 感知：LOD 0-1 跳过细部操作，LOD 2-3 执行完整序列

## 六、总结

当前 CityPhase 的单体建筑生成停留在 **"体量组合"** 阶段——就像乐高积木的堆叠。要让建筑真正具有"设计感"，必须引入 **"构成语法"** 的概念。

**核心结论**：

1. **建筑不是随机生成的，而是遵循语法生成的**。每种风格都有其内部的"句法规则"。
2. **风格 = 参数化规则集 (PRS)**。将设计风格抽象为可调参数，是算法化的关键。
3. **构成语法引擎是下一步的核心基础设施**。它将取代目前散落在各 `gen_*` 函数中的硬编码逻辑。
4. **最终目标**：用户输入的不是"宽 24m、高 120m"，而是"我想一个轻盈的、有悬挑的、包豪斯风格的办公楼"。
