## 下一阶段：v2.0 单体建筑构成语法引擎

### 方向

从"参数化体量"升级为"风格化构成"。当前单体建筑生成器只能调整宽高、退台、扭转等物理参数，无法表达建筑风格的设计意图。v2.0 的目标是让建筑生成器理解**包豪斯、构成主义、极简主义、后现代、粗野主义**等风格的设计法则，并将其转化为可编程的构成语法。

### 核心文档

📄 [`单体建筑生成研究.md`](单体建筑生成研究.md) — 完整的设计蓝本，包含：
- 五种建筑风格的设计法则拆解
- 构成语法的通用框架（谓词语法 + PRS 参数化规则集）
- 从研究到代码实现的四阶段路线图
- 关键算法问题与解决方案

### 四阶段实现路线

```
Phase 1: 构成语法引擎 (Composition Engine)
  ├── 实现谓词语法 API：PLACE / EXTRUDE / INSET / BOOLEAN / SUBDIVIDE / CONNECT / MATERIAL / ALIGN / OFFSET
  └── 封装为 CompositionBuilder 类，替代硬编码的 gen_* 函数

Phase 2: 风格化单体生成器
  ├── bauhaus_gen.py       — 包豪斯：正交网格 + 自由平面 + 流动空间
  ├── constructivist_gen.py — 构成主义：体块交叉 + 对角线动力 + 不对称
  ├── minimalist_gen.py    — 极简主义：纯粹几何 + 片墙 + 光切割
  ├── postmodern_gen.py    — 后现代：历史符号引用 + 矛盾并置 + 色彩
  └── brutalist_gen.py     — 粗野主义：巨型体量 + 雕塑立面 + 粗粝材质

Phase 3: 混合风格与插值
  ├── interpolate_prs(style_a, style_b, ratio) — 风格间的参数化融合
  └── 例：bauhaus × brutalist = 比例控制 + 雕塑感

Phase 4: 设计意图自然语言接口
  ├── 关键词解析："轻盈的、透明的" → constructivist + minimalist
  ├── 关键词解析："厚重的、纪念性的" → brutalist + monumental
  └── 关键词解析："有趣的、彩色的" → postmodern + high_saturation
```

### 关键技术决策

| 问题 | 方案 |
|------|------|
| 体块布尔运算稳定性 | 凸多面体手动布尔 + 失败时退化为包围盒合并 |
| 风格一致性 vs 随机性 | 定义"风格不变量"（必须遵守）和"风格变量"（允许波动） |
| BMesh 性能 | 延迟求值（指令列表）+ 几何缓存 + LOD 感知 |
| 材质表达 | 每种风格定义专属材质节点组（混凝土/玻璃/钢/灰泥） |

### 预期成果

完成 v2.0 后，CityPhase 将具备：

1. **5 种风格化单体生成器**，每种都有独特的构成逻辑和视觉语言
2. **风格混合与插值**，生成"介于两种风格之间"的建筑
3. **设计意图接口**，用户输入关键词而非物理参数
4. **构成语法引擎**，成为可扩展的基础设施，未来可轻松添加新风格
